Experiments & Audit Wall

backtest 가 좋다는 이유로 운영하지 않았다. 6 종류 실험으로 edge 가 통계적으로 살아있는지, sizing 은 어디서 깨지는지, 어디서 calibration 이 무너지는지 — 모두 빨간불 하나라도 있으면 운영 폭을 좁혔다.

Experiment 01 · Bootstrap CI

edge 가 통계적으로 0 이 아닌가

5,000 회 bootstrap. distribution_beta 와 setup_momentum baseline 의 TP-tiered Sharpe diff 분포. 점추정 +0.18, p2.5 -0.05, p97.5 +0.42, 0 위 92.9% — 양수 우세지만 95% CI 하단이 살짝 음수 = marginal evidence.

Sharpe diff distribution (n=5,000 bootstrap)
distribution_beta − setup_momentum, TP-tiered (TP3/TP5 mix). 0 위 92.9%, mean +0.181, median +0.179.

해석: 점추정 (mean +0.181) 양수 + 92.9% boot 가 0 위. 그러나 95% CI 하단이 -0.05 로 0 을 살짝 걸침. 통계적으로는 "양수 가능성 높지만 reject H₀ 는 marginal". 운영 결정: full-size 채택 X, 1/4-1/8 fractional sizing 으로 보수적 운영.

Lessons

10 가지 교훈 — 다음 프로젝트로 가져갈 것

전부 incident 또는 audit 결과. backtest 만 잘 나오면 운영 단계에서 비싼 수업료를 낸다.

01
Label 정의는 학습 metric 이 아니라 사용자 행동 가능성 기준
6-class softprob 은 cross-entropy 잘 떨어졌지만 "5-10% class 60%" 가 매매 결정으로 안 이어짐. h2/h5/h6 처럼 "4h 안 +3% 이상" 같은 actionable label 로 재정의 후 ledger 가 양수로.
02
Train ↔ Live mismatch 는 backtest 에서 안 보인다
Pre-open daily features 는 학습 데이터에서 fully closed 였지만 live 08:55 에서 5min partial. 양쪽 다 closed 인 backtest 는 mismatch 안 잡음. 15m precursor only 로 좁힘.
03
단일 점수 → 조건부 분포로 가면 사용자 판단이 가능해진다
"≥20% tail 후보" 한 줄은 timing 모호. h2/h6/h5 + setup 분해하니 알림이 결정 트리 — h2 강 = 즉발, h5 만 강 = tail bonus 관찰.
04
Calibration 안 된 raw probability 는 거짓말 표시
XGBoost predict_proba 0.85 ≠ "85% 확률". h2 top bucket calib error 32.7pp 측정됨. BucketCalibrator 로 historical hit % 만 표시.
05
Bootstrap CI 가 0 을 걸치면 fractional sizing
TP5 path-aware Sharpe diff = +0.18, 95% CI [-0.05, +0.42]. 점추정 양수지만 통계적으로 marginal. full-size X, 1/4-1/8 sizing 으로 보수.
06
Latency 는 운영 spec, backtest spec 아니다
초기 4m 48s. 08:55 발사면 알림이 09:00 이후 도착 = trigger 후 = 의미 없음. recent-window load 로 21s 단축. 정확도 0.001 보다 latency 30× 단축이 더 큰 P&L 기여.
07
Decision-window guard 가 ledger 를 살린다
Manual rerun, systemd catch-up, debugging 시 stale data 로 ledger 가 오염된 incident 가 실제로 있었음. is_preopen_window() / is_decision_window() + --allow-late-* escape hatch 만 명시 manual 인정.
08
진입가 표기는 사소한 UX 가 아니라 close-out 정합성 도구
알림에 진입가 ≈ 일봉 close 표시. paper ledger 의 entry_price_proxy 와 매칭 → close-out 시 실제 첫 15m bar 와 비교해서 slippage / signal-quality 자동 추적. 한 줄 추가가 디버깅 인프라.
09
Silence 는 기능, 침묵 안 하려고 룰을 늘리지 말 것
detector_v1 의 C8-C10 rank fallback ("threshold 통과 0건이면 top-1 알림") 모두 EV 음수. silence-heavy 가 옳음. cap 과 threshold 를 우선하고 BTC regime 은 알림 문맥으로 표시.
10
Premature complexity 회피 — v1 은 단순, v2 는 데이터로 결정
Multi-scale ablation 에서 h5 가 daily+1h+15m 우위였지만 v1 은 daily-only 채택. 1주 live 결과로 ground truth 잡고 v2 swap 결정. 검증 안 된 복잡성을 운영에 넣지 않음.
