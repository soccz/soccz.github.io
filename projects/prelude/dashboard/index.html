ABOUT · METHODOLOGY

어떻게 만들었나

포트폴리오·발표용 정합성 확보. signal pipeline / 검증 split / 거래비용 가정 / 한계까지 먼저 명시한다 — 결과보다 메서드가 먼저 신뢰를 만든다.

SIGNAL
두 채널 알림
distribution (09:05) — multi-head XGBoost: P(+3% in 4h) / P(+5% in 24h) / P(+20% tail in 24h) + setup library S01·S02·S03. composite = h2 + h6 + 5×h5, top-K 알림.
pre-open (08:55) — 15m precursor 19 features 로 첫 15m / 1h 안 +3% / +5% 즉발 펌프 head. 같은 composite ranking.
VALIDATION
학습 / 검증 위생
label next-day OHLC, market 별 shift(-1) (코인 간 leak 방지). split 5-fold purged Walk-Forward + 10일 embargo + 180일 holdout. threshold chronological OOF p99.95 — 라이브 quantile 재계산 X. look-ahead leak / universe 시간정합성 / 거래비용 차감은 양보 X 위생.
EXECUTION ASSUMPTION
진입·청산·비용
entry KST 09:00 시초가 (sim). 사용자 실 매매는 09:00~09:10 시장가 — 별도 추적.
exit 5% 도달 즉시 익절, 아니면 horizon close (dist 24h / pre 1h). 자동 SL 없음 (사용자 수동 판단).
cost 왕복 0.15% (수수료 0.1% + 슬리피지 0.05%). thin-orderbook 코인은 underestimate 가능.
METRIC
정량 지표
Sharpe / Sortino = 일별 net PnL × √252. Calmar = annualized return / |MDD|. Profit Factor = Σ wins / |Σ losses|. Expectancy = 평균 per-trade. Bootstrap 1000 iter trade-level resample → 95% CI.

한계 — 솔직 명시

이 dashboard 의 결과는 절대 평가에 쓰일 수 없다. 약점을 *내가 먼저* 인정.

01. 표본 너무 작음 — closed n ≈ 28 / 24, 일자별 aggregation 시 unique day n 가 한 자리 수. Sharpe / Sortino 의 standard error 가 매우 큼. bootstrap 95% CI 가 0 을 가로질러 통계적 유의 X. 30+ 거래가 쌓일 때까지 결과는 *지표 모양만* 으로 본다.
02. survivorship bias 미처리 — pyupbit 의 현재 listing 만 가져옴. 과거 상폐 코인이 panel 에서 빠져 historical EV 가 부풀 수 있음. Upbit 공시 archive 로 historical universe 재구성 필요.
03. 거래비용 0.15% 가정의 낙관성 — ≥20% tail pump 코인은 호가 sparse → 실제 진입 시 시장 충격 + 슬리피지 0.5~1.5% 가 일반. cost 모델을 ATR / volume 기반으로 강화할 필요.
04. regime-dependent — BTC bull regime 에서만 동작 설계. bear 구간은 침묵. 전체 시장 cycle 을 cover 하지 못하므로 누적 결과는 시기 의존성 큼.
05. 5% TP 룰의 단순성 — sweep 결과 대신 사용자 의사결정에 맞춘 룰. avg_max +6.82% 가 5% TP 에서 잘려 +20% tail 을 놓침. 룰 민감도 분석 (3% / 7% / no TP) 추후 추가.
