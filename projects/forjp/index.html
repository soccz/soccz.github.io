Overview

두 종류의 사람을 위한 데이트 플래너

"오늘 뭐 할까?"라는 질문에 답하는 방식은 사람마다 다르다. 어떤 사람은 대충 말하면 알아서 짜줬으면 하고, 어떤 사람은 직접 고르고 순서도 바꾸고 싶다. 그 두 스타일을 각각 다른 UI 흐름으로 만드는 게 이 프로젝트의 출발점이었다.

P mode — 대화형 플래닝
챗봇이 몇 가지를 물어보고 코스를 만들어준다. 장소, 시간, 분위기, 예산만 말하면 5가지 테마의 코스 후보가 나온다. 즉흥적인 사람에게 맞는 방식이다.
J mode — 직접 편집
추천 결과를 놓고 장소를 바꾸고, 순서를 조정하고, 동선 최적화 제안도 받는다. 꼼꼼하게 계획하는 사람이 쓰기 좋은 방식이다.
날씨 & 상황 인식
서울 실시간 날씨를 가져와서 비 오는 날엔 실내 코스를 우선 추천한다. 단순한 기능이지만, "지금 이 순간"을 반영하려는 시도다.
P Mode

대화로 코스를 만든다

복잡한 입력 폼 대신 대화로 정보를 모으는 방식이 더 자연스럽지 않을까? 라는 질문에서 시작했다. 챗봇이 단계별로 질문하고, 수집된 정보로 후보 장소들을 가져온 다음, 같은 데이터에서 성격이 다른 5가지 코스를 뽑아낸다.

Plan Variant 1 — Efficient
이동 시간 최소 코스
"같은 장소들인데 순서가 다르면 이동 시간이 크게 달라진다. 최근접 이웃 알고리즘으로 동선을 최적화한다."
루트 최적화이동 시간 최소
Plan Variant 2 — Mood / Romantic
분위기 우선 코스
"quietScore, visualScore로 장소의 감성 점수를 계산하고, 분위기에 맞는 장소를 우선 선발한다."
감성 점수vibe 매칭
Plan Variant 3 — Value / Relaxed
가성비 & 여유 코스
"valueDensity = stayMinutes / (cost / 1000). 같은 돈으로 더 오래 머물 수 있는 장소를 선호한다."
가성비 점수체류 시간예산 최적화
J Mode

직접 고르고, 직접 다듬는다

추천 결과를 그냥 받아들이는 게 아니라 직접 손볼 수 있어야 한다는 생각이었다. 장소를 교체하고, 순서를 바꾸고, "이 동선이 더 효율적이에요"라는 제안도 받을 수 있다. 그리고 그 과정에서 왜 이 장소가 추천됐는지 — 품질 필터가 어떻게 작동했는지를 보여주려 했다.

Feature 1 — Alt Panel
장소 교체
"각 슬롯마다 대안 후보를 미리 계산해둔다. 교체하면 이동 시간과 총 비용이 즉시 재계산된다."
대안 후보즉시 재계산swap
Feature 2 — Quality Filter
장소 품질 필터링
"카테고리 불일치, 과도한 거리, 중복 상호명, 체인점을 자동으로 걸러낸다. 왜 이 장소가 걸러졌는지 추적 가능하다."
카테고리 필터거리 필터중복 제거체인 감점
Feature 3 — Route Optimizer
동선 최적화 제안
"현재 순서보다 이동 시간을 줄일 수 있는 순서가 있으면 제안한다. 최근접 이웃 알고리즘 기반."
TSP 근사순서 제안이동 단축
Architecture

API 키가 없어도 개발할 수 있게

외부 API 의존도가 높은 프로젝트일수록 개발 중 막히는 순간이 많다. Provider 패턴으로 데이터 소스를 추상화하면, API 키가 없어도 mock 데이터로 전체 흐름을 테스트할 수 있다. live / mock 전환은 환경 변수 하나로 된다.

Provider 계층
장소 검색(Kakao), 대중교통(ODsay), 챗(mock/live), 리뷰 요약 각각에 독립 provider가 있다. 환경 변수가 없으면 자동으로 mock fallback한다.
live
전부 실데이터
hybrid
일부 실데이터 + fallback
mock
전부 목업 fallback

날씨 캐시 TTL
POST /api/chat
대화 단계(step)와 수집된 정보(collected)를 받아 다음 AI 응답을 반환한다. Provider 교체로 실제 LLM과 mock 응답을 전환할 수 있다.
GET /api/weather
Open-Meteo에서 서울 현재 날씨를 가져와 30분 캐시로 반환한다. 비/눈 여부를 실내 코스 우선순위 결정에 사용한다.
POST /api/recommendations
장소 후보, 캐시 상태, provider 상태, 시간대 혼잡도 진단, 대안 후보를 한 번에 반환한다.
GET /api/shared-plans/:id
저장된 코스를 링크로 공유할 수 있다. Supabase에 없으면 로컬 fallback으로 응답한다.
GET /api/diagnostics
어떤 provider가 활성화돼 있는지, 환경 변수 설정 이슈가 있는지 확인할 수 있다.
POST /api/custom-plan
사용자가 직접 선택한 장소 목록을 실제 플래너 구조로 변환한다. J mode 코스 확정 시 호출된다.
