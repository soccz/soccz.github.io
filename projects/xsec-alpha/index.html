Thesis

왜 이걸 만들었는가

gan_t에서 1년 동안 CVAE, GAN, MC-Dropout, 35개 피처를 쌓았지만 결과는 "스크리너보다 못한 모델"이었다. 근본 원인을 진단하고, 완전히 다른 접근으로 다시 시작했다.

gan_t: 왜 실패했나
절대 수익률 예측은 BTC 베타에 종속된다. 3h 호라이즌의 AC=0.025는 노이즈 수준이고, train이 test를 100% 포함하고 있었다. confidence head는 상수 0.503을 출력했고, 98% short 편향에 short 차단까지 걸려서 실행 가능한 추천이 0건이었다.
IC = -0.06 Sharpe 0.17 Screener > Model
xsec_alpha: 뭘 바꿨나
절대 수익률 → residual return (coin - beta × BTC). 개별 코인 예측 → 크로스섹션 상대 랭킹. per-market MinMax → per-timestamp z-score. fillna(0) → ffill only + 480행 drop. train⊃test → temporal holdout 20%.
IC = +0.19 Holdout PASS Dual Model

Long Model Journey

Long 모델의 진화 — v1에서 v3까지

short이 short 전문이라면, long은 완전히 다른 문제로 봐야 한다. 이 과정에서 "크립토에서 momentum은 어디서도 안 먹힌다"는 핵심 교훈을 얻었다.

Long v1FAILED
Momentum/Trend 접근 — 전면 실패
가설: "short이 역추세라면, long은 순추세(momentum)로 가야 한다."

momentum_12h, momentum_24h, breakout_24h, volume_surge, bullish_order_flow, relative_strength 팩터로 학습. Holdout IC +0.177 — 겉으로는 좋았다.

하지만 20+ 에이전트 다각도 감사에서 근본 결함 발견:

발견 1: 모멘텀 IC가 6h~72h 전 구간에서 음수. Bull 레짐에서도 mean-reversion이 지배적.

Factor DiscoveryBREAKTHROUGH
"조용해진 코인이 터진다" — range_contraction 발견
10개 대안 팩터를 bull 레짐 + top100에서 탐색. range_contraction_12h가 IC=+0.179(24h)로 기존 모든 팩터보다 강했다. 4분기 split에서 전부 양수, rolling 7d 최솟값도 양수. 운이 아님.

vol_inv_24h(IC=+0.168)도 강했지만 탈락. 이유: range_contraction과 r=0.785로 강하게 상관 — 둘 다 "조용한 코인"을 측정하지만, range_contraction이 IC가 더 높고 고저폭 기반이라 return std와 다른 정보를 담는다.
Horizon Decision12h
왜 24h가 아니라 12h인가
range_contraction은 24h에서 IC +0.179로 최강이지만, 24h 리밸런스로 바꾸면 holdout 평가점이 22개 → 5~6개로 급감해 t-stat 신뢰도가 불가능. 12h는 IC +0.137로 충분하면서 평가점을 유지하는 타협점이다. 신호 강도보다 검증 가능성을 우선했다.

Lessons Learned

핵심 교훈

01
크립토에서 bull이라도 단기 모멘텀은 mean-reversion에 밀린다
6h~72h 전 구간, bull regime에서도 모멘텀 IC는 전부 음수. "올라간 코인을 사라"는 크립토에서 작동하지 않는다. 대신 "조용해진 코인이 터진다"가 맞다.
02
모델을 나눴으면 long은 short의 반대편이 아니라 완전히 다른 문제다
v1은 short 철학(reversal)의 변형판을 long에 얹었다. Ridge가 자동으로 부호를 뒤집어서 겉으로는 작동했지만, 설계 의도와 실제 동작이 불일치. v3에서 "compression + reversal"로 재정의하니 설계=결과가 일치.
03
IC가 좋아도 실행에서 검증해야 한다
v1 holdout IC +0.177이었지만 long hit rate 43.6%, 수익의 90%가 short. "L/S IC"와 "long-only 성능"은 다른 질문이다. 반드시 long-only, short-only 분리 검증 필수.
04
pred_std가 낮은 건 정규화 문제가 아니라 피처 예측력 한계
Ridge alpha를 1.0→0.01로 바꿔도 계수가 동일. R²=1%. 모델이 "정직하게 신호가 없다고 말하고 있는 것"이었다. 정규화 탓이 아님.
05
멀티 에이전트 토론이 사람의 편향을 잡는다
혼자였으면 v1의 IC +0.177에 만족하고 배포했을 것. 13개 에이전트가 "팩터 IC 전부 음수", "long hit rate 43%", "pred_std 근상수"를 동시에 지적해서 근본 재설계로 이어졌다.
