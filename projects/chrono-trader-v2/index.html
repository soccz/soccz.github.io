Overview

무엇을 만들려 했나

암호화폐 시장에서 '지금 오르는 코인'과 '다음에 오를 코인'은 연결되어 있다는 가설에서 출발했다. 이 두 관점을 하나의 모델에 담고, 불확실성을 측정하며, 매일 자동으로 돌아가는 시스템을 목표로 했다.

⬡
패턴 전이 탐지
BTC/ETH에서 시작된 상승이 알트코인으로 퍼지는 리드-래그 구조를 포착한다. 과거 유사 패턴(메모리 뱅크)과 현재 시퀀스의 유사도로 "지금 어디에 있는지"를 판단한다.

2026 Jan — Phase 1
데이터 파이프라인 수정 + 12h 전환
세 가지 근본 문제를 발견했다: ① fillna(0)이 RSI=0, volatility=0 같은 불가능한 값을 만들었다. ② Train/Test가 100% 겹쳐 모든 메트릭이 무효였다. ③ 3h 예측 AC=0.025(노이즈 수준), 12h AC=0.100(통계적으로 유의미). 호라이즌을 12h로 전환하고 ffill/bfill + 480행 드롭, temporal holdout 20%를 적용했다.

2026 Feb — Phase 2
Heteroscedastic 디코더
CVAE 디코더를 이종 분산(heteroscedastic) 구조로 교체했다. μ와 log σ를 별도 헤드로 출력해 불확실성이 예측마다 다르게 계산되도록 설계. 하지만 훈련 후 log_sigma_mean = -5.05(초기화 편향에 갇힘), σ ≈ 0.0067로 사실상 상수. KL collapse도 확인(KL=0.000 throughout 100 epoch).
log_sigma 상수 수렴 — 설계 실패
2026 Mar — Phase 3
추천 필터 수정 + 이상 예측 차단
라이브 추천에서 세 문제 발견: ① direction_consistency 임계값 0.66이 너무 엄격해 추천이 거의 없었다 → 0.55로 완화. ② TOSHI predicted_return=71×, POKT=9-11×(불확실성 폭발) → 이상치 하드 필터 추가. ③ Short 신호에서 gate < 0.25이면 예측력 없음 → gate_value 필터 추가. 세 필터 모두 uncertainty bypass 이전에 적용했다.

Lessons

무엇을 배웠나

1년간의 반복 학습-검증 사이클에서 실제로 유효했던 교훈들. 알고 있던 것을 확인한 것도 있고, 실제로 틀렸던 것도 있다.

1
평가 지표부터 잡아야 한다 — IC는 5분, 백테스트는 22시간
22시간 걸리는 백테스트를 반복하는 동안 신호 자체가 있는지 없는지를 확인하지 못했다. Spearman IC는 5분 만에 신호 품질을 알려준다. 빠른 피드백 루프 없이는 잘못된 방향으로 긴 시간을 쓸 수 있다. np.corrcoef(pred, actual)가 먼저였어야 했다.
2
승률은 시장 베타를 제거하고 봐야 한다
Short 58.3% 승률을 모델 신호로 착각할 뻔했다. BTC가 -21.9% 하락한 기간이었다. 방향이 맞은 것이지 예측이 맞은 것이 아니다. 알파를 측정할 때는 반드시 시장 수익률 대비 상대 수익률로 봐야 한다. 크로스섹션 접근법이 이 문제를 구조적으로 해결한다.
3
데이터 파이프라인 버그가 모델 성능보다 먼저다
fillna(0)은 RSI=0, volatility=0 같은 물리적으로 불가능한 값을 만든다. Train/Test가 겹치면 모든 메트릭이 낙관적으로 편향된다. beta의 bfill은 미래 누출을 만든다. 모델을 복잡하게 만들기 전에 데이터가 깨끗한지부터 확인해야 한다.
4
운영 인프라는 연구보다 먼저 완성해야 한다
처음부터 systemd 타이머, watchdog, freshness gate, 텔레그램 알림을 갖춘 운영 파이프라인을 만든 것은 올바른 선택이었다. 모델 성능이 부족하더라도 시스템이 매일 돌아가면서 실제 데이터로 문제를 발견할 수 있었다. "완벽한 모델을 만들고 배포"가 아니라 "돌아가는 것을 먼저 만들고 개선"이 실질적이다.
5
복잡한 모델이 단순한 스크리너를 이기지 못할 수 있다
GAN+Transformer 앙상블(53.1%)이 단순 스크리너(55.8%)에 졌다. 모델 복잡도를 높이는 것보다 신호 자체의 예측력을 확인하는 것이 우선이다. 12h OHLCV만으로는 크립토 수익률 예측에 충분한 정보가 없을 수 있다. 더 나은 피처가 더 복잡한 모델보다 효과적이다.
6
필터 시스템이 나쁜 신호를 구제할 수 있다
raw 예측의 Short IC=-0.158이지만 10단계 퍼널을 통과한 거래의 IC=+0.18. 나쁜 예측들을 걸러내는 필터 자체가 알파를 만들 수 있다. 다만 거래 수가 너무 적어 통계적으로 불안정하다. 필터를 통과하는 거래가 충분히 많아야 의미가 있다 — 커버리지와 품질의 트레이드오프.
