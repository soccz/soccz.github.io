이번 주 이야기

10,000명의 학생 이름과 점수가 있습니다.
"지민"의 점수를 찾아야 합니다. 

# 방법 1: 리스트 — 처음부터 찾아야 함
students_list = [("지민", 95), ("수진", 87), ..., ("Zara", 72)]
for name, score in students_list:
    if name == "지민":
        print(score)
# 지민가 운 나쁘게 맨 뒤에 있으면 10,000번 비교!

# 방법 2: 딕셔너리 — 이름만 알면 즉시
students_dict = {"지민": 95, "수진": 87, ..., "Zara": 72}
print(students_dict["지민"])   # 딱 1번 — 데이터가 100만 개여도 똑같이 빠름

딕셔너리는 어떻게 100만 개에서도 1번만에 찾을 수 있을까요?
비밀은 해시(Hash)입니다 — 값을 "주소"로 변환하는 함수.

이번 주 핵심 질문:
  • 왜 리스트는 O(n)인데 딕셔너리는 O(1)인가?
  • key는 왜 반드시 hashable(불변)이어야 하는가?
  • {}는 왜 빈 딕셔너리고 빈 세트는 set()인가?
  • Python 딕셔너리는 왜 삽입 순서를 보장하는가? (3.7+)

왜 딕셔너리가 필요한가?

비유: 사물함 vs 줄 서기

줄 서기(리스트) vs 사물함(딕셔너리):
리스트는 번호표를 뽑고 줄 서는 것입니다. "3번"을 찾으려면 1번, 2번, 3번 순서대로 지나가야 합니다.
딕셔너리는 이름이 붙은 사물함입니다. "지민"라고 적힌 사물함은 번호 계산으로 직접 찾아갑니다.

compact dict (Python 3.6+, Raymond Hettinger 제안):
기존 dict는 (hash, key, value) 튜플 배열에 빈 슬롯이 많아 메모리를 낭비했습니다.
compact dict는 두 배열로 분리합니다:

  • dk_indices: 해시 테이블 슬롯 → 엔트리 인덱스 (소형 배열, 슬롯당 1~8 bytes)
  • dk_entries: (me_hash, me_key, me_value) 엔트리를 빈 슬롯 없이 밀착 배열

이로 인해 삽입 순서 자연 보존(엔트리가 순서대로 추가)과 메모리 절약(최대 ~50%)이 동시에 달성됩니다.

Part B — Associative Array ADT와 Python dict의 복잡도

Associative Array는 키-값 쌍을 저장하고 key → value 매핑을 지원하는 ADT입니다. 구현 방법에 따라 복잡도가 달라집니다:

구현탐색삽입삭제순서
정렬 배열 + 이진 탐색O(log n)O(n)O(n)정렬 순서
BST (균형 미보장)O(n) 최악O(n) 최악O(n) 최악키 순서
Red-Black Tree (Java TreeMap)O(log n)O(log n)O(log n)키 순서
Hash Table (Python dict)O(1) 평균O(1) 평균O(1) 평균삽입 순서

해시 테이블의 O(1)은 평균(average-case)입니다 — 해시 충돌이 적다는 전제 하에 대부분의 연산이 O(1)입니다. 단, resize(테이블 크기 확장)는 O(n)이 발생하지만 이는 상각(amortized) O(1)로 분석됩니다. 최악의 경우(모든 키가 같은 슬롯에 충돌) 단일 연산이 O(n)이 될 수 있습니다.

해시의 원리 — 왜 O(1)인가

초급

비유: 도서관 분류번호

도서관 분류번호 비유:
도서관에서 "파이썬 프로그래밍" 책을 찾으려면:
· 리스트: 1번 서가부터 순서대로 전부 확인
· 딕셔너리: 제목을 계산기에 넣으면 → "프로그래밍 분야 → 205호 → 3번째 칸" 즉시 알려줌

이 "계산기"가 hash() 함수입니다. 
# hash() 함수: 어떤 값이든 → 정수 "주소"로 변환
print(hash(42))       # 42 (정수는 자기 자신)
print(hash(0))        # 0
print(hash(True))     # 1
print(hash(False))    # 0

# 같은 값은 항상 같은 해시 (같은 실행 내에서)
print(hash("hello") == hash("hello"))   # True — 일관성 보장
print(hash((1, 2, 3)) == hash((1, 2, 3)))  # True
실행 결과:
42 0 1 0 True True 
# 해시가 주소가 되는 원리 (단순화)
table_size = 8   # 딕셔너리 내부에 8개 슬롯 (시작)

key = "지민"
slot = hash(key) % table_size   # 해시 → 슬롯 번호 계산

print(f'hash("지민") % 8 = {hash("지민") % 8}')   # 0~7 사이 숫자
# → 이 슬롯 번호에 직접 저장하고, 직접 읽음 → O(1)!
실행 결과 (실행마다 다름 — hash randomization):
hash("지민") % 8 = 3

hash() 함수의 성질 / hashable vs unhashable

# 정수의 해시: 자기 자신 (CPython 특성)
for n in [0, 1, 2, 100, -5]:
    print(f"hash({n}) = {hash(n)}")

# 특별 케이스: hash(-1) = -2
# CPython에서 -1은 C 레벨에서 "오류" 신호로 예약됨
# → hash()는 절대 -1을 반환하지 않음 (-1이 나오면 -2로 대체)
print(hash(-1))   # -2
print(hash(-2))   # -2  (둘 다 -2)

# 값이 같으면 해시도 같아야 함 (등호 일관성)
print(hash(1) == hash(1.0) == hash(True))   # True — 모두 정수 1
print(1 == 1.0 == True)                     # True — 이므로 해시도 같아야 함
실행 결과:
hash(0) = 0 hash(1) = 1 hash(2) = 2 hash(100) = 100 hash(-5) = -5 -2 -2 True True 
# hashable 여부 확인: __hash__ 속성 존재 여부
print(hash.__doc__[:50] if hash.__doc__ else "")

# Mutable 객체는 unhashable — 변경되면 해시가 달라질 수 있으니
try:
    hash([1, 2, 3])   # 리스트: Mutable → unhashable
except TypeError as e:
    print(f"list: {e}")

try:
    hash({"a": 1})    # 딕셔너리: Mutable → unhashable
except TypeError as e:
    print(f"dict: {e}")

try:
    hash({1, 2, 3})   # 세트: Mutable → unhashable
except TypeError as e:
    print(f"set: {e}")

# frozenset: 불변 세트 → hashable!
print(hash(frozenset({1, 2, 3})))   # (정수값)
실행 결과:
list: unhashable type: 'list' dict: unhashable type: 'dict' set: unhashable type: 'set' (frozenset의 해시값 — 정수)
해시 랜덤화 (Hash Randomization):
Python 3.3+부터 문자열·바이트·datetime의 해시는 실행마다 달라집니다 (PYTHONHASHSEED=random).
이유: 공격자가 해시 충돌을 의도적으로 유발하는 DoS 공격 방지 (Python 3.3 기본 활성화, PEP 456에서 SipHash 알고리즘으로 교체).
정수·float의 해시는 항상 일정합니다.

해시 함수의 조건 / 커스텀 __hash__ / 딕셔너리 조회 흐름

# 좋은 해시 함수의 3가지 조건:
# 1. 결정적 (Deterministic): 같은 입력 → 항상 같은 출력
# 2. 균등 분포 (Uniform): 출력이 슬롯에 고르게 퍼짐
# 3. 빠름 (Fast): O(1)에 가깝게 계산

# 딕셔너리 조회 흐름 (d["key"]의 내부):
# 1. hash("key") → 정수 h 계산
# 2. slot = h & mask (mask = table_size - 1)
# 3. dk_indices[slot]의 엔트리 인덱스 → dk_entries[i] 확인
# 4. 저장된 hash와 key가 일치하면 → value 반환
# 5. 불일치(충돌)이면 → 다음 슬롯으로 (perturbation probing)

# 커스텀 __hash__ 구현
class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __eq__(self, other):
        if not isinstance(other, Point):
            return NotImplemented
        return self.x == other.x and self.y == other.y

    def __hash__(self):
        # tuple의 해시를 재활용 — Python 권장 패턴
        return hash((self.x, self.y))

    def __repr__(self):
        return f"Point({self.x}, {self.y})"

p1 = Point(1, 2)
p2 = Point(1, 2)
p3 = Point(3, 4)

print(p1 == p2)           # True
print(hash(p1) == hash(p2))   # True (같은 값 → 같은 해시)
print(hash(p1) == hash(p3))   # False (보통 다른 값 → 다른 해시, 하지만 충돌 가능 **)

Part B — 해시 함수 이론: 완전 해시 vs 보편 해시

완전 해시 함수 (Perfect Hash Function): n개의 키에 대해 충돌이 전혀 없는 해시 함수. 키 집합이 미리 알려진 경우에만 구성 가능 (예: Python 내장 키워드 집합).

보편 해시 (Universal Hashing): 임의의 키 집합에 대해 충돌 확률이 1/m 이하 (m=테이블 크기)가 되도록 랜덤하게 선택된 해시 함수 군. Python의 PYTHONHASHSEED 기반 랜덤화는 이 아이디어를 구현합니다.

비둘기집 원리 (Pigeonhole Principle): 키 공간(무한)이 슬롯 수(유한)보다 크므로 충돌은 수학적으로 불가피합니다. 해시 테이블은 충돌을 없애는 것이 아니라 처리하는 방법을 설계합니다.

Part B — 상각 분석 (Amortized Analysis)

딕셔너리 삽입은 대부분 O(1)이지만, 테이블이 꽉 차면(load factor ≥ 2/3) resize가 발생합니다. resize는 O(n)이지만 n번의 삽입마다 한 번씩 발생합니다.

상각 O(1) 증명 (회계 방법, Accounting Method):
각 삽입 시 실제 비용 1 + "저축" 2를 누적합니다 (총 3크레딧). resize 시 n개를 재배치하는 비용 n은 누적된 n×2 크레딧으로 지불합니다. 따라서 삽입 1회의 상각 비용 = O(1).

Part A — CPython perturbation probing 공식 / DKIX_EMPTY·DKIX_DUMMY

# CPython Objects/dictobject.c의 탐사 공식 (Python으로 재현)
def cpython_probe_sequence(hash_val, mask):
    """CPython dict의 충돌 해결 탐사 순서를 시뮬레이션."""
    perturb = hash_val
    i = hash_val & mask          # 초기 슬롯

    visited = [i]
    for _ in range(min(mask, 20)):  # 최대 20 단계만
        perturb >>= 5              # perturbation 감소
        i = (i * 5 + perturb + 1) & mask   # 다음 슬롯
        visited.append(i)
        if len(set(visited)) == len(visited):
            pass  # 중복 없이 탐색 중
        else:
            break   # 순환 완료

    return visited

# mask = table_size - 1 (table_size는 2의 거듭제곱)
mask = 7  # table_size = 8

print("hash=0 탐사 순서:", cpython_probe_sequence(0, mask))
print("hash=3 탐사 순서:", cpython_probe_sequence(3, mask))
실행 결과:
hash=0 탐사 순서: [0, 1, 6, 7, 4, 5, 2, 3] hash=3 탐사 순서: [3, 0, 1, 6, 7, 4, 5, 2]

슬롯의 상태는 세 가지입니다:

  • DKIX_EMPTY: 한 번도 사용된 적 없는 슬롯. 탐사 중 이 슬롯을 만나면 → 키 없음 확정.
  • Active: 현재 키-값이 저장된 슬롯.
  • DKIX_DUMMY (Tombstone): 삭제된 슬롯. 탐사를 계속해야 함 (중간에 멈추면 다음 충돌 키를 못 찾음).

from collections import Counter

def analyze_text(text: str) -> dict:
    """텍스트 분석 — 10주 개념 총동원.

    사용된 개념:
    - 변수/타입: text, words, result
    - 조건문: if not text
    - 함수: def, return, 타입 어노테이션
    - for 반복문: Iterator Protocol
    - 리스트/세트: words, unique_words
    - 딕셔너리: result, Counter
    - 클로저 없이도 함수가 독립적으로 작동
    """
    if not text:        # Truthiness (2주)
        return {}

    words = text.lower().split()          # str 메서드 (2주)
    unique_words = set(words)             # 세트 (10주)
    word_count = Counter(words)           # 딕셔너리 (10주)

    result = {                            # 딕셔너리 컴프리헨션 (10주)
        "total": len(words),
        "unique": len(unique_words),
        "top3": word_count.most_common(3),
        "avg_length": sum(len(w) for w in words) / len(words),  # 제너레이터 (7주)
    }
    return result

sample = "the quick brown fox jumps over the lazy dog the"
result = analyze_text(sample)
for key, value in result.items():        # items() 순회 (10주)
    print(f"{key}: {value}")

class GoodPoint:
    def __init__(self, x, y):
        self.x, self.y = x, y
    def __eq__(self, other):
        return (self.x, self.y) == (other.x, other.y)
    def __hash__(self):
        return hash((self.x, self.y))

def top_words(text, n=3):
    if not text:
        return []
    counts = {}
    for word in text.lower().split():

            counts[word] = counts.get(word, 0) + 1
    return sorted(counts.items(), key=lambda x: x[1], reverse=True)[:n]

def _probe(self, key):
    return hash(key) % self.size

def __setitem__(self, key, value):
    slot = self._probe(key)
    first_deleted = None
    for _ in range(self.size):
        if self.slots[slot] is self.EMPTY:
            insert_slot = first_deleted if first_deleted is not None else slot
            self.slots[insert_slot] = key
            self.values[insert_slot] = value
            self.count += 1
            if self.count / self.size > 2/3:
                self._resize()
            return
        elif self.slots[slot] is self.DELETED:
            if first_deleted is None:
                first_deleted = slot
        elif self.slots[slot] == key:
            self.values[slot] = value
            return
        slot = (slot + 1) % self.size

def __getitem__(self, key):
    slot = self._probe(key)
    for _ in range(self.size):
        if self.slots[slot] is self.EMPTY:
            raise KeyError(key)
        elif self.slots[slot] is not self.DELETED and self.slots[slot] == key:
            return self.values[slot]
        slot = (slot + 1) % self.size
    raise KeyError(key)

def __delitem__(self, key):
    slot = self._probe(key)

딕셔너리 (Dictionary)

Key-Value 쌍의 집합. Python 3.7+부터 삽입 순서 보장. key는 hashable, value는 제한 없음.

Key-Value 쌍

딕셔너리의 기본 단위. {"name": "지민"}에서 "name"이 key, "지민"가 value.

해시 함수 (Hash Function)

임의의 데이터를 고정 크기 정수로 변환하는 함수. Python의 hash(). 결정적·균등·빠름.

hashable

__hash__()가 정의되어 hash 값이 있고 생애 동안 변하지 않는 객체. dict key/set 원소로 사용 가능.

unhashable

hash 값이 없거나 변할 수 있는 객체. list, dict, set. key로 사용 시 TypeError 발생.

해시 테이블 (Hash Table)

해시 함수로 키를 배열 인덱스로 변환해 저장하는 자료구조. 평균 O(1) 탐색.

해시 충돌 (Hash Collision)

서로 다른 두 키의 해시값이 같은 슬롯을 가리키는 현상. 비둘기집 원리로 불가피.

오픈 어드레싱 (Open Addressing)

충돌 시 배열 내 다른 빈 슬롯을 찾아 저장하는 충돌 해결 방식. Python dict/set 사용.

Perturbation Probing

CPython의 해시 충돌 탐사 알고리즘. i = (5*i + perturb + 1) & mask. 군집화 방지.

부하 인수 (Load Factor)

사용 중인 슬롯 / 전체 슬롯. Python dict는 2/3 초과 시 테이블 자동 확장(resize).

compact dict

Python 3.6+의 딕셔너리 내부 구조. dk_indices(해시 테이블)와 dk_entries(엔트리 배열) 분리. Raymond Hettinger 제안.

뷰 객체 (View Object)

.keys(), .values(), .items()의 반환값. 복사본이 아닌 딕셔너리의 실시간 뷰.

KeyError

존재하지 않는 key를 d[key]로 접근할 때 발생. .get(key, 기본값)으로 방지 가능.

.get(key, default)

key가 없으면 default를 반환 (기본 None). KeyError 없이 안전하게 접근.

.setdefault(key, default)

key가 없을 때만 default로 추가. 있으면 기존 값 유지. 그루핑 패턴에 유용.

세트 (set)

순서 없고 중복 없는 hashable 객체의 집합. in 연산 O(1). 빈 세트는 set().

frozenset

불변 세트. hashable → dict key/set 원소로 사용 가능. set처럼 집합 연산 지원.

합집합 (Union)

a | b. a 또는 b에 있는 모든 원소. a.union(b)와 동일.

교집합 (Intersection)

a & b. a와 b 모두에 있는 원소. a.intersection(b)와 동일.

차집합 (Difference)

a - b. a에 있지만 b에 없는 원소. a.difference(b)와 동일.

대칭 차집합 (Symmetric Difference)

a ^ b. a 또는 b 중 하나에만 있는 원소. a.symmetric_difference(b)와 동일.

딕셔너리 컴프리헨션

{k: v for k, v in ...}. dict를 간결하게 생성. 자체 스코프를 가짐 (Python 3).

세트 컴프리헨션

{x for x in ...}. set을 간결하게 생성. 중복 자동 제거.

defaultdict

collections.defaultdict. 없는 key에 접근 시 기본값 자동 생성. 카운팅·그루핑에 유용.

Counter

collections.Counter. 빈도 카운팅 전용 딕셔너리. .most_common(n)으로 상위 n개 조회.

O(1) 상각 (Amortized O(1))

대부분의 연산은 O(1)이지만 resize 시 O(n) 발생. 연산 전체를 평균 내면 O(1)로 수렴.

Tombstone (DKIX_DUMMY)

삭제된 슬롯을 표시하는 마커. 빈 슬롯과 구별하여 탐사 체인이 끊기지 않도록 함.

hash randomization

Python 3.3+: 문자열 해시가 실행마다 달라짐 (PYTHONHASHSEED). DoS 해시 충돌 공격 방지.

Separate Chaining

해시 충돌 시 같은 슬롯에 연결 리스트를 연결하는 방식. Java HashMap이 사용. Open Addressing과 대비.

TypedDict

typing.TypedDict. 런타임엔 일반 dict지만 mypy 등 정적 분석기가 구조를 검사. AI 코드 검증에 유용.

# 지금까지 배운 것: 절차적 프로그래밍 (Procedural Programming)
def analyze_data(data):
    """함수로 묶고, 딕셔너리로 관리하고, 세트로 중복 제거."""
    unique = set(data)
    count  = {item: data.count(item) for item in unique}
    return count

# 다음 단계 1: 클래스 (Class) — 데이터 + 함수를 묶는 더 큰 단위
class DataAnalyzer:
    def __init__(self, data):
        self.data = data              # 인스턴스 변수
        self._cache = {}              # 내부 상태

    def count(self):
        """각 항목의 빈도 반환."""
        if not self._cache:
            for item in self.data:
                self._cache[item] = self._cache.get(item, 0) + 1
        return self._cache

# 다음 단계 2: 예외 처리 (Exception Handling)
try:
    result = int("abc")   # ValueError!
except ValueError as e:
    print(f"변환 실패: {e}")
finally:
    print("항상 실행됨")   # 정리 코드

# 다음 단계 3: 파일 I/O — with문으로 자원 관리
with open("data.txt", "w", encoding="utf-8") as f:
    f.write("파이썬은 왜?\n")
# with 블록 끝에서 파일 자동 닫힘 (__exit__ 호출)

# 다음 단계 4: 제너레이터 — 메모리 효율적 이터레이터
def fibonacci():
    """무한 피보나치 수열 생성기."""
    a, b = 0, 1
    while True:
        yield a       # yield: 값을 반환하고 일시정지
        a, b = b, a + b

fib = fibonacci()
print([next(fib) for _ in range(8)])   # [0, 1, 1, 2, 3, 5, 8, 13]
