이번 주 이야기

지난 주 for 루프로 리스트를 순회했습니다. 그런데 이런 상황은 어떨까요? 

# 사용자에게 숫자를 입력받아 합산 — 언제 끝날지 모름
numbers = []
# for i in range(???):   ← 몇 번 반복할지 모름!
#     numbers.append(int(input()))

for는 미리 알고 있는 Iterable을 순회합니다. 하지만 "사용자가 'quit'을 입력할 때까지", "조건이 거짓이 될 때까지"처럼 끝을 미리 알 수 없는 반복에는 for가 맞지 않습니다. 

# while: "이 조건이 참인 동안 계속"
total = 0
while True:
    text = input("숫자 입력 (quit으로 종료): ")
    if text == "quit":
        break
    total += int(text)
print(f"합계: {total}")

while은 조건을 보면서 반복합니다. Iterator가 없어도 됩니다. 상태만 있으면 충분합니다. 오늘은 while 루프를 바이트코드까지 완전히 분해합니다.

왜 while이 존재하는가? — Iterable 없이 반복하기

초급

for와 while: 두 가지 반복의 차이

비유로 이해하기:
for = 택배 기사 — "이 목록에 있는 집을 순서대로 방문한다". 목록이 있어야 시작.
while = 경비원 — "문 앞에서 계속 확인한다. 조건이 참이면 통과, 거짓이면 닫음". 목록 없이 조건만으로 동작.

for: "이 목록을 순서대로 처리해줘" — 목록이 있어야 함
while: "이 조건이 참인 동안 계속해줘" — 조건만 있으면 됨 

# for: 항목이 정해져 있을 때
fruits = ["사과", "바나나", "딸기"]
for fruit in fruits:
    print(fruit)   # 3번 실행, 끝이 명확

# while: 언제 끝날지 조건으로 결정
count = 0
while count < 3:
    print(f"{count}번째")
    count += 1     # 이 줄 없으면 영원히 반복!
실행 결과:
0번째 1번째 2번째
while의 규칙: 조건 변수를 루프 안에서 반드시 바꿔야 합니다. 안 바꾸면 조건이 영원히 True → 무한 루프!
중급

for가 안 되는 상황 — while이 필요한 이유

while이 필요한 실제 상황들:
  • 게임 루프: "플레이어가 살아있는 동안" 계속 실행 — for로는 끝을 알 수 없음
  • 네트워크 재시도: "연결이 성공할 때까지" 계속 시도
  • 사용자 입력 검증: "올바른 입력이 들어올 때까지" 계속 받기
  • 데이터 스트림 처리: "파일의 끝(EOF)을 만날 때까지" 계속 읽기
  • 알고리즘 수렴: "오차가 충분히 작아질 때까지" 계속 반복
# for가 불가능한 상황: 끝을 모를 때
# 네트워크 재시도 패턴
import random

def try_connect():
    return random.random() > 0.7   # 30% 성공률

attempts = 0
while not try_connect():
    attempts += 1
    print(f"연결 실패 ({attempts}회 시도), 재시도...")

print(f"연결 성공! (총 {attempts+1}회 시도)")
# ① 입력 횟수를 미리 모를 때
password = ""
while password != "correct":
    password = input("비밀번호: ")
print("로그인 성공")

# ② 수렴 조건 (미리 몇 번인지 모름)
x = 100.0
while abs(x * x - 2) > 1e-10:   # 오차가 충분히 작아질 때까지
    x = (x + 2 / x) / 2          # 뉴턴법으로 √2 근사
print(f"√2 ≈ {x:.10f}")         # 1.4142135624

# ③ 이진 탐색 (범위가 줄어드는 조건)
arr = [1, 3, 5, 7, 9, 11, 13]
target = 7
left, right = 0, len(arr) - 1
while left <= right:
    mid = (left + right) // 2
    if arr[mid] == target:
        print(f"인덱스 {mid}")
        break
    elif arr[mid] < target:
        left = mid + 1
    else:
        right = mid - 1

세 경우 모두 "몇 번 반복할지"가 아니라 "언제 멈출지"가 핵심입니다. 이때 while이 for보다 자연스럽습니다.

고급

Python 공식 문서의 while 문법

Python 공식 문서(reference/compound_stmts)의 while 문 정의: 

while_stmt ::= "while" assignment_expression ":" suite
               ["else" ":" suite]

조건 자리에 assignment_expression이 옵니다 — 일반 표현식뿐 아니라 Walrus 연산자(:=)도 허용합니다. 

import re

text = "price:100 count:5 discount:20"
pos = 0
results = {}

# := (Walrus)로 매칭과 할당을 동시에
while m := re.match(r'(\w+):(\d+)', text[pos:]):
    key, val = m.group(1), int(m.group(2))
    results[key] = val
    pos += m.end()
    if pos < len(text) and text[pos] == ' ':
        pos += 1   # 공백 건너뜀

print(results)   # {'price': 100, 'count': 5, 'discount': 20}
실행 결과:
{'price': 100, 'count': 5, 'discount': 20}
while과 for의 근본 차이: for는 __iter__ → __next__라는 Protocol을 따릅니다. while은 Protocol이 없습니다 — 표현식이 Truthy인 동안 실행될 뿐입니다.
전공자

Part A — while vs for 바이트코드 골격 비교

import dis

def f_for(lst):
    for x in lst:
        pass

def f_while(n):
    while n > 0:
        n -= 1

dis.dis(f_for)
print("---")
dis.dis(f_while)
f_for 핵심 바이트코드 (Python 3.12):
LOAD_FAST lst GET_ITER ← iter(lst) 호출 — Iterator 생성 >> FOR_ITER N ← next() + StopIteration 처리 STORE_FAST x JUMP_BACKWARD → FOR_ITER >> END_FOR
f_while 핵심 바이트코드 (Python 3.12):
>> LOAD_FAST n ← 조건 변수 로드 (매 반복마다) LOAD_CONST 0 COMPARE_OP > POP_JUMP_IF_FALSE → END ← False이면 루프 탈출 LOAD_FAST n LOAD_CONST 1 BINARY_OP (-=) STORE_FAST n JUMP_BACKWARD → LOAD_FAST ← 조건 검사로 되돌아감 >> (END) RETURN_CONST None

핵심 차이:

  • for: GET_ITER(1회) + FOR_ITER(매 반복) — Iterator가 상태를 관리
  • while: 조건 표현식을 매 반복마다 직접 평가 — POP_JUMP_IF_FALSE로 탈출
  • for는 Iterator에 위임, while은 스스로 조건 평가 — 근본적으로 다른 메커니즘

Part B — 왜 for가 while보다 Iterable 순회에 효율적인가

for x in lst:를 while로 구현하면: 

i = 0
while i < len(lst):   # len(lst) 매 반복 평가
    x = lst[i]
    i += 1

이 버전은 매 반복마다 len(lst) 호출 + 인덱싱 비용이 발생합니다. for 루프의 FOR_ITER_LIST 특화는 이보다 빠릅니다 (Week07 §03 참조). 반면 while은 조건 표현식이 단순 비교일 때 오버헤드가 적어 수렴 조건, 이진 탐색, 상태 변화 같은 복잡한 탈출 조건에 적합합니다.

while 기본 문법 — 조건 루프의 구조

초급

비유: while은 경비원이다

while 루프를 경비원(Guard)이라고 생각하세요.
경비원은 문 앞에 서서 매번 확인합니다: "조건이 True면 통과, False면 멈춤."

경비원 비유:
while 조건: → "조건이 참인 동안 통과시켜라"
    실행할 코드
→ 코드가 끝나면 경비원에게 다시 돌아가 재확인
→ 조건이 False가 되는 순간 문이 닫힘

while 조건 설계 3단계

while을 처음 짤 때 이 순서대로 생각하세요:

  1. "언제 계속할까?" → 조건 작성 while count <= 5:
  2. "무엇을 할까?" → 루프 body 작성
  3. "조건이 언젠가 False가 되는가?" → 상태 변경 확인 count += 1
③을 빠뜨리면 무한 루프! 작성 후 반드시 "이 루프는 언젠가 끝나는가?"를 확인하세요.

Walrus 연산자 := 와 while

Python 3.8+의 Walrus 연산자(:=)는 while 조건에서 할당과 검사를 동시에 수행합니다. 반복적인 패턴을 제거합니다: 

import sys

# ❌ 반복적인 패턴 (조건과 할당이 두 곳)
chunk = sys.stdin.buffer.read(4096)
while chunk:
    process(chunk)
    chunk = sys.stdin.buffer.read(4096)   # 중복!

# ✅ Walrus로 단순화
while chunk := sys.stdin.buffer.read(4096):
    process(chunk)

# 파일 읽기에도 동일하게 적용
with open("data.txt") as f:
    while line := f.readline():   # 빈 문자열("")은 Falsy
        print(line.strip())

# 정규식 매칭
import re
text = "abc123def456"
pos = 0
while m := re.search(r'\d+', text[pos:]):
    print(m.group())   # 123, 456
    pos += m.end()
Walrus 주의: 파일의 readline()은 EOF에서 빈 문자열 ""을 반환합니다. ""는 Falsy이므로 while line := f.readline():이 자연스럽게 종료됩니다.

import dis

실행 결과 (Python 3.12):
1 RESUME 0 2 >> LOAD_FAST 0 (n) ← 매 반복마다 n 로드 LOAD_CONST 1 (0) COMPARE_OP 4 (>) POP_JUMP_IF_FALSE 6 (to 22) ← False이면 종료로 jump 3 LOAD_FAST 0 (n) LOAD_CONST 2 (1) BINARY_OP 23 (-=) STORE_FAST 0 (n) JUMP_BACKWARD 5 (to 4) ← 조건 평가로 되돌아감 >> RETURN_CONST 0 (None)

>>가 붙은 두 위치: LOAD_FAST(루프 시작, 조건 평가 지점)와 RETURN_CONST(루프 종료 후). for와 달리 Iterator 없이 COMPARE_OP + POP_JUMP_IF_FALSE로 조건을 직접 처리합니다.

Part B — Walrus 연산자의 바이트코드

def walrus_while(it):
    while x := next(it, None):
        pass

dis.dis(walrus_while)
# 핵심 바이트코드:
# LOAD_GLOBAL next
# LOAD_FAST it
# LOAD_CONST None
# CALL 2
# COPY 1          ← := 대입을 위해 스택 복사
# STORE_FAST x    ← x에 할당
# POP_JUMP_IF_FALSE → end  ← None(Falsy)이면 종료
# (body)
# JUMP_BACKWARD → LOAD_GLOBAL next

Walrus(:=)는 표현식 평가 결과를 COPY한 후 STORE_FAST로 변수에 할당하고, 원본 값을 조건 검사에 사용합니다. 할당과 검사가 단일 표현식 안에서 처리됩니다.

함정: while x := next(it, None):는 None뿐 아니라 0, "", False, [] 등 모든 Falsy 값에서 조기 종료됩니다. 이터레이터가 Falsy 값을 산출할 수 있다면 while (x := next(it, SENTINEL)) is not SENTINEL: 패턴을 사용하세요 (§06 Sentinel 패턴 참조).

while True + break — 무한 루프의 올바른 설계

초급

"일단 시작하고 중간에 판단"하는 루프

while True:는 "무조건 반복, 탈출은 break로"를 뜻합니다. 종료 조건이 루프 중간에 있을 때 사용합니다. 

# 숫자 맞추기 게임
import random
secret = random.randint(1, 10)
attempts = 0

while True:
    guess = int(input("1~10 사이 숫자를 맞춰보세요: "))
    attempts += 1
    if guess == secret:
        print(f"{attempts}번 만에 맞췄습니다!")
        break
    elif guess < secret:
        print("더 큰 숫자입니다.")
    else:
        print("더 작은 숫자입니다.")

while attempts < 100:처럼 쓸 수도 있지만, "정답을 맞출 때까지"라는 의도는 while True + break가 더 명확합니다.

중급

탈출 조건 설계 원칙 — break는 하나만

# ❌ 여러 break: 탈출 경로가 많아 추적이 어려움
while True:
    data = input()
    if data == "quit":
        break
    if data == "exit":
        break
    if data == "q":
        break
    process(data)

# ✅ 탈출 조건을 하나로 통합
QUIT_COMMANDS = {"quit", "exit", "q"}
while True:
    data = input()
    if data in QUIT_COMMANDS:
        break
    process(data)

# ✅ 복잡한 경우: is_done() 함수로 분리
def is_done(data, attempts, budget):
    return data == "quit" or attempts > 10 or budget <= 0

attempts = 0
budget = 100
while True:
    data = input()
    attempts += 1
    if is_done(data, attempts, budget):
        break
    budget -= int(data)
설계 원칙: break가 여러 개라면 "탈출 조건이 함수로 분리될 만큼 복잡하다"는 신호입니다.

def find_prime_pair(limit):
    """합이 limit인 소수 쌍 탐색"""
    def is_prime(n):
        if n < 2: return False
        for i in range(2, int(n**0.5) + 1):
            if n % i == 0: return False
        return True

    a = 2
    while a < limit:
        b = limit - a
        if is_prime(a) and is_prime(b):
            return (a, b)   # return이 중첩 루프를 즉시 탈출
        a += 1
    return None

Part A — while True 바이트코드 최적화

Python 3.12의 flowgraph 최적화기는 while True:의 조건 (True)이 항상 참임을 컴파일 타임에 인식하고, 조건 평가 코드를 제거합니다: 

import dis

def with_condition(n):
    while n > 0:         # 조건 평가 필요
        n -= 1

def with_true():
    x = 0
    while True:          # 조건 평가 불필요
        x += 1
        if x >= 5:
            break

dis.dis(with_condition)
print("---")
dis.dis(with_true)
with_condition — 조건 평가 있음:
>> LOAD_FAST n LOAD_CONST 0 COMPARE_OP > POP_JUMP_IF_FALSE → END ← 매 반복마다 평가 ...body... JUMP_BACKWARD → LOAD_FAST >> END
with_true — 조건 평가 없음 (최적화):
LOAD_CONST 0 ← x = 0 초기화 STORE_FAST x >> LOAD_FAST x ← 바로 body 시작, 조건 체크 없음! LOAD_CONST 1 BINARY_OP (+=) STORE_FAST x LOAD_FAST x LOAD_CONST 5 COMPARE_OP (>=) POP_JUMP_IF_FALSE → after_if ← x < 5: break 안 함 → if 블록 건너뜀 JUMP_FORWARD → END ← break: 루프 탈출 >> after_if: JUMP_BACKWARD → >> ← 정상 반복: 루프 시작으로 >> END

while True:에서 LOAD_GLOBAL True + POP_JUMP_IF_FALSE가 완전히 제거됩니다. while True는 while 조건 중 가장 효율적입니다.

Part B — CPython flowgraph.c의 Peephole Optimization (dead code elimination)

CPython의 Python/flowgraph.c에서 while True:는 두 단계로 최적화됩니다: 

1단계 (AST): While(test=Constant(True), ...)
2단계 (flowgraph): test가 항상 True → JUMP_IF_FALSE 제거
   → 루프 body + JUMP_BACKWARD만 남음

반면 while bool_var:
  → 런타임에 bool_var 값을 알 수 없어 최적화 불가
  → 매 반복마다 LOAD_FAST + POP_JUMP_IF_FALSE 실행

이것이 while True: ... if cond: break 패턴이 while not_done:보다 빠를 수 있는 이유입니다 — 탈출 조건을 while 헤더가 아닌 body 안에서 평가하므로, 컴파일러가 while 헤더를 최적화 제거할 수 있습니다.

import dis

            break

        print("not found")

핵심 바이트코드 구조 (Python 3.12):
>> LOAD_FAST n COMPARE_OP (>) POP_JUMP_IF_FALSE → else_block ← 조건 False → else 실행 (body: n == target 검사) POP_JUMP_IF_FALSE → JUMP_BACKWARD JUMP_FORWARD → after_else ← break: else 건너뜀 >> JUMP_BACKWARD → LOAD_FAST >> else_block: (print "not found") >> after_else: RETURN_CONST None

Part A — Sentinel과 StopIteration의 관계

Python의 Iterator Protocol은 소진 신호로 StopIteration을 사용합니다. 이것 자체가 "프로토콜 정의된 Sentinel" 패턴입니다: 

from collections.abc import Iterator

# StopIteration = Iterator Protocol의 공식 sentinel
class CountUpTo:
    def __init__(self, limit):
        self.n = 0
        self.limit = limit
    def __iter__(self): return self
    def __next__(self):
        if self.n >= self.limit:
            raise StopIteration   # ← 공식 sentinel
        val = self.n
        self.n += 1
        return val

# next(it, default): default가 비공식 sentinel
it = CountUpTo(3)
DONE = object()
while (val := next(it, DONE)) is not DONE:
    print(val)   # 0, 1, 2

from enum import Enum, auto

class ParseState(Enum):
    FIELD_START = auto()
    IN_FIELD = auto()
    IN_QUOTED = auto()
    AFTER_QUOTE = auto()

def parse_csv_field(text):
    """간단한 CSV 필드 파서"""
    state = ParseState.FIELD_START
    fields = []
    current = []

    for char in text + ",":   # 끝에 , 추가로 마지막 필드 처리
        if state == ParseState.FIELD_START:
            if char == '"':
                state = ParseState.IN_QUOTED
            elif char == ',':
                fields.append("".join(current))
                current = []
            else:
                current.append(char)
                state = ParseState.IN_FIELD

        elif state == ParseState.IN_FIELD:
            if char == ',':
                fields.append("".join(current))
                current = []
                state = ParseState.FIELD_START
            else:
                current.append(char)

        elif state == ParseState.IN_QUOTED:
            if char == '"':
                state = ParseState.AFTER_QUOTE
            else:
                current.append(char)

        elif state == ParseState.AFTER_QUOTE:
            if char == ',':
                fields.append("".join(current))
                current = []
                state = ParseState.FIELD_START

    "RUNNING":    {"pause": "PAUSED",  "stop":  "IDLE"},
    "PAUSED":     {"resume": "RUNNING", "stop": "IDLE"},

def run_state_machine(initial: str, events: list[str]) -> str:
    state = initial
    while events:
        event = events.pop(0)
        if event in TRANSITIONS.get(state, {}):
            state = TRANSITIONS[state][event]
        else:
            raise ValueError(f"Invalid transition: {state} + {event}")
    return state

def binary_search(arr, target):
    """정렬된 배열에서 target 탐색 — O(log n)"""
    left, right = 0, len(arr) - 1

    # for가 안 되는 이유:
    # 반복 횟수가 탐색 결과에 따라 달라짐 (left, right가 동적으로 변함)
    # "left <= right"라는 상태 조건이 핵심

        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1   # 못 찾음

def sqrt_newton(n, tolerance=1e-10):
    """뉴턴-랩슨법으로 √n 계산 — 수렴할 때까지 반복"""
    if n < 0:
        raise ValueError("음수의 제곱근은 실수가 아닙니다")
    if n == 0:
        return 0.0

    x = n / 2.0   # 초기 추정값

    # for는 불가: 몇 번 반복해야 할지 모름 (수렴 속도가 n에 따라 다름)
    while True:
        x_new = (x + n / x) / 2
        if abs(x_new - x) < tolerance:   # 수렴 조건
            return x_new
        x = x_new

Part B — 루프 불변식(Loop Invariant)과 정확성 증명

루프 불변식(Loop Invariant)은 각 반복 시작 시점에서 항상 참인 조건입니다 (반복 도중에는 일시적으로 깨질 수 있으며, 반복 끝에 복원됩니다). 올바른 알고리즘 설계의 핵심 도구이며, while 루프의 정확성을 증명하는 데 사용됩니다. 

def binary_search(arr, target):
    """
    루프 불변식: arr[low] <= target <= arr[high] (target이 존재한다면)
    초기: low=0, high=n-1 → 전체 배열
    유지: 매 반복에서 절반을 버리되 불변식 유지
    종료: low > high → target 없음, 또는 arr[mid] == target → 발견
    """
    low, high = 0, len(arr) - 1

    while low <= high:
        mid = (low + high) // 2    # 중간 인덱스

        if arr[mid] == target:
            return mid             # 발견!
        elif arr[mid] < target:
            low = mid + 1          # 오른쪽 절반만 탐색
        else:
            high = mid - 1         # 왼쪽 절반만 탐색

    return -1   # 발견 못함

arr = [1, 3, 5, 7, 9, 11, 13, 15]
print(binary_search(arr, 7))    # 3 (인덱스)
print(binary_search(arr, 6))    # -1 (없음)

# 왜 이 알고리즘이 항상 올바른가?
# 불변식 증명:
# 1. 초기: low=0, high=7 → 모든 원소 포함 ✓
# 2. 유지: mid를 버리고 반쪽으로 줄여도 불변식 유지 ✓
# 3. 종료: low > high → 빈 범위 → target 없음 ✓
print(f"이진 탐색으로 O(log n) 달성: 8개 원소, 최대 {len(arr).bit_length()}번 비교")
루프 불변식의 3가지 조건:
1. 초기화(Initialization): 루프 시작 전 불변식이 참
2. 유지(Maintenance): 각 반복 후에도 불변식이 참
3. 종료(Termination): 루프 종료 시 불변식 + 종료 조건으로 알고리즘 정확성 증명

    while left < right:

def factorize(n):
    factors = []
    divisor = 2
    while divisor * divisor <= n:
        while n % divisor == 0:
            factors.append(divisor)
            n //= divisor
        divisor += 1
    if n > 1:
        factors.append(n)   # 남은 n은 소수

Sentinel 패턴 — object() 동작 분석

object() sentinel의 메모리와 동일성(identity) 동작을 분석하세요:

import sys, dis

SENTINEL = object()

def consume_with_sentinel(data):
    it = iter(data)
    results = []
    while (val := next(it, SENTINEL)) is not SENTINEL:
        results.append(val)
    return results

# 1. SENTINEL의 메모리 특성
print(f"SENTINEL 크기: {sys.getsizeof(SENTINEL)} bytes")
print(f"SENTINEL id: {id(SENTINEL)}")

# 2. None이 sentinel이 될 수 없는 상황
data = [1, None, 3, None, 5]
print(consume_with_sentinel(data))   # [1, None, 3, None, 5]

# 3. SENTINEL 동일성 분석
s1 = object()
s2 = object()
print(f"s1 is s2: {s1 is s2}")   # False
print(f"s1 == s2: {s1 == s2}")   # False
print(f"hash(s1): {hash(s1)}, hash(s2): {hash(s2)}")   # 다름

# 4. 바이트코드에서 SENTINEL is not val 비교
dis.dis(consume_with_sentinel)
# IS_OP opcode: is/is not 비교 전용 바이트코드 확인

무한 루프 방어 래퍼 구현

while 루프에 최대 반복 횟수를 강제하는 안전 래퍼를 구현하고 성능을 분석하세요:

import dis, timeit, functools

def safe_loop(max_iterations=10_000):
    """while 루프를 데코레이터로 감싸는 안전 래퍼"""
    def decorator(condition_fn):
        @functools.wraps(condition_fn)
        def wrapper(*args, **kwargs):
            iterations = 0
            while condition_fn(*args, **kwargs):
                iterations += 1
                if iterations >= max_iterations:
                    raise RuntimeError(
                        f"루프가 {max_iterations}회 초과 — 무한 루프 의심"
                    )
            return iterations
        return wrapper
    return decorator

# 사용 예시
count = [0]

@safe_loop(max_iterations=100)
def converge():
    count[0] += 1
    return count[0] < 50

iters = converge()
print(f"정상 종료: {iters}회")

용어 사전 (22개)

while 문 (while statement)

조건 표현식이 Truthy인 동안 suite를 반복 실행하는 제어 구조. Iterator 없이 동작.

while True

항상 참인 조건의 무한 루프. Python 컴파일러가 조건 평가 코드를 최적화 제거. break로만 탈출.

while-else

while 루프가 break 없이 조건이 False가 되어 종료할 때 else 블록 실행. "no break" 패턴.

Sentinel (파수꾼 값)

루프 종료를 알리는 특별한 값. 정상 데이터와 절대 혼동되지 않아야 함. None 또는 object().

object()

전 우주에서 유일한 새 객체 생성. is 비교로만 동일성 구분. 완벽한 sentinel 값.

State Machine (상태 기계)

유한한 상태 집합과 전이 규칙으로 동작을 표현하는 계산 모델. while 루프로 자연스럽게 구현.

Walrus 연산자 (:=)

Python 3.8+. 표현식 내에서 할당과 평가를 동시에 수행. while 조건에서 코드 중복 제거.

do-while 시뮬레이션

Python에는 do-while 없음. while True + break로 "먼저 실행 후 조건 검사"를 구현.

이진 탐색 (Binary Search)

정렬된 배열에서 O(log n) 탐색. left ≤ right 상태 조건으로 while 루프가 적합.

뉴턴-랩슨법

수치 해석에서 방정식의 근을 반복으로 근사. 수렴 조건 기반 while 루프의 대표 사례.

수렴 조건 (Convergence)

반복 알고리즘에서 결과가 충분히 안정될 때까지 반복. abs(new - old) < tolerance.

무한 루프 (Infinite Loop)

탈출 조건이 없거나 조건 변수가 업데이트되지 않아 영원히 실행되는 루프.

POP_JUMP_IF_FALSE

while 조건이 False일 때 루프 끝으로 jump하는 바이트코드. for의 FOR_ITER와 대비.

JUMP_BACKWARD

while 루프 body 실행 후 조건 검사 위치로 되돌아가는 바이트코드. for에서도 사용.

Peephole Optimization (핍홀 최적화)

바이트코드 수준에서 불필요한 명령어를 제거하는 최적화. while True의 조건 평가 코드 제거(dead code elimination)에 적용.

루프 불변식 (Loop Invariant)

각 반복 시작 시점에서 항상 참인 조건 (반복 도중 일시적으로 깨질 수 있으며, 반복 끝에 복원). 이진탐색: "target이 있다면 arr[left..right] 안에 있다".

정지 문제 (Halting Problem)

임의의 프로그램이 반드시 종료되는지 판별하는 것이 불가능함. 튜링(1936) 증명.

iter(callable, sentinel)

callable을 반복 호출해 sentinel과 같은 값이 나오면 StopIteration. 이중 인수 iter() 형태.

Guard Clause + while

루프 초반 continue/break로 특수 케이스를 먼저 처리. 중첩 줄이기 패턴 (Week07 §08 참조).

Enum (열거형)

State Machine에서 상태를 명명하는 데 사용. 오타 방지, IDE 자동완성, 가독성 향상.

전이 테이블 (Transition Table)

State Machine의 상태 × 이벤트 → 다음 상태 매핑. 딕셔너리로 표현. 코드 없이 규칙 추가 가능.

수렴 알고리즘 vs 순회 알고리즘

수렴: while로 구현 (몇 번인지 모름). 순회: for로 구현 (Iterable 있음). 선택 기준.

def binary_search(arr, target):
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target: return mid
        elif arr[mid] < target: left = mid + 1
        else: right = mid - 1
    return -1
