클로저 & 데코레이터
— 함수를 감싸는 함수

함수는 데이터입니다 — 변수에 담고, 인수로 넘기고, 반환할 수 있습니다.
클로저(Closure)는 함수가 자신의 탄생 환경을 배낭처럼 메고 다닙니다.
데코레이터(Decorator)는 기존 함수를 수정 없이 선물 포장합니다.
@property, @classmethod, @lru_cache — 파이썬의 핵심 문법이 모두 데코레이터입니다.

01 — 이번 주 이야기

로그인한 사용자만 접근할 수 있는 API가 있습니다. 함수가 100개라면, 각 함수마다 "로그인 확인 코드"를 100번 복사해야 할까요?

아닙니다. 파이썬에는 함수를 감싸는 방법이 있습니다. 

# 이게 전부입니다
@login_required
def get_user_data(user_id):
    return db.query(user_id)

@login_required
def update_profile(user_id, data):
    db.update(user_id, data)

@login_required는 두 함수에 "로그인 확인" 기능을 추가했습니다. 원본 함수는 단 한 줄도 수정하지 않고서.

이것이 데코레이터입니다. 그리고 데코레이터를 이해하려면 먼저 클로저를 이해해야 합니다. 클로저를 이해하려면 먼저 함수도 데이터라는 사실을 이해해야 합니다.

이번 주는 그 순서대로 갑니다.

02 — First-Class Function: 함수도 데이터다

초급

먼저, 불편함을 느껴봅시다

숫자 리스트를 여러 방식으로 변환해야 합니다:

numbers = [1, 2, 3, 4, 5]

# 두 배로 만들기
doubled = []
for n in numbers:
    doubled.append(n * 2)

# 제곱하기
squared = []
for n in numbers:
    squared.append(n ** 2)

# 문자열로 바꾸기
stringified = []
for n in numbers:
    stringified.append(str(n))

세 번 모두 같은 구조(빈 리스트 → for → append)인데 "어떤 변환을 적용할지"만 다릅니다. "변환 방법"을 인수로 넘길 수 있다면 한 번만 쓸 수 있지 않을까요? 

def transform(numbers, func):
    return [func(n) for n in numbers]

print(transform(numbers, lambda n: n * 2))   # [2, 4, 6, 8, 10]
print(transform(numbers, lambda n: n ** 2))  # [1, 4, 9, 16, 25]
print(transform(numbers, str))               # ['1', '2', '3', '4', '5']

func에 함수를 넘겼습니다. 이것이 가능하려면 함수가 값이어야 합니다.

함수를 변수에 넣을 수 있다

파이썬에서 함수는 특별한 존재가 아닙니다. 숫자, 문자열처럼 값(Value)입니다.

def greet(name):
    return f"안녕, {name}!"

# 함수를 변수에 저장
say_hello = greet

# say_hello는 이제 greet과 같은 함수
print(say_hello("철수"))   # 안녕, 철수!
print(greet is say_hello)  # True — 같은 객체!
흔한 혼동: say_hello = greet는 함수를 복사하는 게 아닙니다. 같은 함수 객체를 두 이름으로 가리키는 것입니다. (12주차의 이름표 개념 — 이름이 객체를 가리킨다)

함수를 다른 함수의 인수로 넘길 수도 있습니다:

def apply(func, value):
    return func(value)

def double(x):
    return x * 2

print(apply(double, 5))   # 10
print(apply(len, "hello")) # 5
왜 이게 중요한가요?
map(int, ["1","2","3"])에서 int에 괄호가 없는 이유가 바로 이것입니다. int 자체가 함수 객체이고, map에 함수를 인수로 전달하는 것입니다. int()는 함수를 호출한 결과(변환된 값)를 넘기는 것으로, 완전히 다릅니다.
중급

함수를 반환할 수도 있다

함수가 함수를 반환할 수 있습니다. 이것이 클로저와 데코레이터의 핵심입니다.

def make_multiplier(n):
    def multiply(x):
        return x * n      # n은 바깥 함수의 변수!
    return multiply       # 함수 객체를 반환

times3 = make_multiplier(3)
times5 = make_multiplier(5)

print(times3(10))   # 30
print(times5(10))   # 50
print(times3(7))    # 21
주목: make_multiplier(3)이 끝났는데도 times3은 n=3을 기억합니다. 어떻게? → 다음 섹션에서 설명합니다.
고급

First-Class Object의 조건

Python에서 함수가 First-Class Object인 증거:

def greet(name): return f"Hi, {name}"

# 1. 변수에 할당 가능
f = greet

# 2. 자료구조에 저장 가능
ops = [len, str.upper, lambda x: x[::-1]]
print([op("hello") for op in ops])  # [5, 'HELLO', 'olleh']

# 3. 함수의 인수로 전달 가능
def apply_twice(func, x):
    return func(func(x))

print(apply_twice(lambda x: x*2, 3))  # 12

# 4. 함수의 반환값이 될 수 있음
def get_operation(op_name):
    if op_name == "double": return lambda x: x*2
    if op_name == "square": return lambda x: x**2

op = get_operation("square")
print(op(4))   # 16

# 5. 속성을 가질 수 있음
greet.version = "1.0"
print(greet.version)   # 1.0
Higher-Order Function (고차 함수): 함수를 인수로 받거나 함수를 반환하는 함수. map, filter, sorted(key=...), functools.reduce가 모두 고차 함수입니다.
전공자

Part A: PyObject로서의 함수 — PyFunctionObject

CPython에서 def는 PyFunctionObject를 생성합니다.

# 함수 객체의 내부 속성 탐색
def add(x, y):
    """두 수를 더한다."""
    return x + y

import inspect

print(type(add))              # <class 'function'>
print(add.__name__)           # add
print(add.__doc__)            # 두 수를 더한다.
print(add.__code__)           # <code object add ...>
print(add.__code__.co_varnames)  # ('x', 'y')
print(add.__code__.co_filename)  # 현재 파일
print(add.__defaults__)       # None (기본값 없음)
print(add.__globals__ is globals())  # True — 전역 스코프 참조
Part A — PyFunctionObject 구조 (CPython): 함수 객체는 func_code(바이트코드), func_globals(전역 딕셔너리 참조), func_closure(자유 변수 셀 배열), func_defaults(기본값 튜플)를 가집니다. MAKE_FUNCTION 바이트코드 명령어가 실행될 때 생성됩니다.

Part B: Lambda Calculus와 First-Class Function

함수를 First-Class로 다루는 개념은 1930년대 Alonzo Church의 Lambda Calculus에서 유래합니다. 모든 계산은 함수 적용(application)과 함수 추상화(abstraction)로 표현 가능하다는 이론입니다. LISP(1958)가 이를 처음 프로그래밍 언어에 도입했고, Haskell, Scheme, Python이 계승했습니다.

Python의 lambda x: x*2 문법은 Lambda Calculus의 λx.x×2 표기를 직접 계승합니다.

03 — 클로저: 배낭을 메고 다니는 함수

핵심 비유: 배낭 (Backpack)

여행을 떠나는 함수가 있습니다. 집(바깥 함수의 스코프)을 떠날 때, 필요한 물건(자유 변수)을 배낭에 챙겨 가져갑니다. 집이 사라져도 배낭 속 물건은 함수와 함께 살아있습니다.

이 배낭이 바로 클로저(Closure)입니다.

초급

클로저 — 기억하는 함수

클로저는 "자신이 만들어진 환경을 기억하는 함수"입니다.

def make_counter():
    count = 0               # 이 변수를 기억합니다

    def counter():
        nonlocal count
        count += 1
        return count

    return counter          # 함수를 반환

c1 = make_counter()
c2 = make_counter()         # 독립적인 카운터

print(c1())   # 1
print(c1())   # 2
print(c1())   # 3
print(c2())   # 1  ← c2는 자신만의 count를 가짐
print(c1())   # 4
make_counter()가 끝났는데도 count는 살아있습니다. c1이 그것을 배낭에 담았기 때문입니다.
중급

nonlocal과 자유 변수

클로저 안에서 바깥 변수를 읽기만 할 때는 그냥 사용 가능합니다. 하지만 수정하려면 nonlocal이 필요합니다. 

# 읽기: nonlocal 불필요
def make_greeter(greeting):
    def greet(name):
        return f"{greeting}, {name}!"   # greeting은 읽기만
    return greet

hello = make_greeter("안녕")
hi = make_greeter("Hi")

print(hello("철수"))   # 안녕, 철수!
print(hi("영희"))     # Hi, 영희!
# 수정: nonlocal 필요
def make_adder(start):
    total = start

    def add(n):
        nonlocal total    # 없으면 UnboundLocalError!
        total += n
        return total

    return add

acc = make_adder(100)
print(acc(10))   # 110
print(acc(20))   # 130
print(acc(5))    # 135
nonlocal 없이 수정하면: 
def make_buggy():
    x = 0
    def inner():
        x += 1    # UnboundLocalError: x는 local로 간주됨
        return x
    return inner
파이썬 컴파일러는 함수 안에 x = 할당이 있으면 x를 지역 변수로 취급합니다. 읽기 전에 할당이 오면 에러가 납니다.

클로저를 확인하는 방법

def outer(x):
    def inner():
        return x * 2
    return inner

f = outer(10)
print(f.__closure__)                          # (<cell at 0x...>,)
print(f.__closure__[0].cell_contents)         # 10  ← 배낭 속 물건!
print(f.__code__.co_freevars)                 # ('x',)  ← 자유 변수 이름
고급

클로저의 고전적 함정: 루프와 클로저

# 버그 버전
funcs = []
for i in range(3):
    funcs.append(lambda: i)   # 모두 같은 i를 참조!

print([f() for f in funcs])   # [2, 2, 2] — 예상: [0, 1, 2]
왜 [2, 2, 2]인가? 람다가 i의 값을 복사하는 게 아니라 i라는 이름을 참조합니다. 루프가 끝나면 i=2이고, 세 람다 모두 같은 i를 바라봅니다. 
# 수정 1: 기본 인수로 값 고정 (가장 관용적)
funcs = [lambda i=i: i for i in range(3)]
print([f() for f in funcs])   # [0, 1, 2]

# 수정 2: 팩토리 함수로 캡처
def make_func(n):
    return lambda: n

funcs = [make_func(i) for i in range(3)]
print([f() for f in funcs])   # [0, 1, 2]
전공자

Part A: Cell 객체와 자유 변수 — CPython 내부

CPython은 클로저를 구현하기 위해 Cell 객체를 사용합니다. 바깥 함수와 안쪽 함수가 같은 Cell 객체를 공유하고, Cell 객체 안에 실제 값이 저장됩니다. 

import dis

def outer():
    x = 42
    def inner():
        return x
    return inner

# outer의 바이트코드에서 MAKE_CELL 확인
dis.dis(outer)
# MAKE_CELL  x   ← x를 cell로 변환

# inner의 자유 변수 확인
f = outer()
print(f.__code__.co_freevars)           # ('x',)
print(f.__closure__[0].cell_contents)   # 42
Part A — Cell 공유 메커니즘: outer의 로컬 프레임과 inner 함수 객체가 동일한 PyCell을 가리킵니다. 따라서 outer가 x를 수정하면 inner도 새 값을 봅니다. Python 3.11에서 MAKE_CELL 명령어가 도입되어 (bpo-43693) 지역 변수를 즉시 cell로 변환합니다.

Part B: 함수형 프로그래밍과 클로저

클로저는 함수형 프로그래밍에서 상태를 캡슐화하는 핵심 도구입니다. 객체지향의 인스턴스 변수(self.x)와 동일한 역할을 순수 함수만으로 구현할 수 있습니다. 

# 클로저로 구현한 카운터 vs 클래스
# 둘은 기능적으로 동일합니다

# 클로저 버전
def make_counter():
    count = 0
    def inc(): nonlocal count; count += 1; return count
    def dec(): nonlocal count; count -= 1; return count
    def get(): return count
    return inc, dec, get

inc, dec, get = make_counter()

# 클래스 버전
class Counter:
    def __init__(self): self.count = 0
    def inc(self): self.count += 1; return self.count
    def dec(self): self.count -= 1; return self.count
    def get(self): return self.count

04 — 데코레이터 기초: 선물 포장

핵심 비유: 선물 포장 (Gift Wrapping)

원본 선물(함수)은 그대로 두고, 예쁜 포장지(wrapper)로 감쌉니다. 받는 사람은 포장지를 뜯으면 원본 선물을 볼 수 있지만, 포장지가 추가 기능(리본, 카드, 메시지)을 더해줍니다.

데코레이터는 함수를 수정하지 않고 기능을 추가하는 선물 포장입니다.

초급

@ 문법으로 함수 꾸미기

def say_hi():
    print("안녕하세요!")

# 데코레이터 없이 — 직접 포장
def shout(func):
    def wrapper():
        print("📣 ", end="")
        func()
        print("(크게 외쳤습니다)")
    return wrapper

say_hi = shout(say_hi)   # 포장
say_hi()
# 📣  안녕하세요!
# (크게 외쳤습니다)

@ 문법은 이것의 축약형입니다:

def shout(func):
    def wrapper():
        print("📣 ", end="")
        func()
        print("(크게 외쳤습니다)")
    return wrapper

@shout
def say_hi():
    print("안녕하세요!")

say_hi()
# 📣  안녕하세요!
# (크게 외쳤습니다)
@shout는 say_hi = shout(say_hi)와 완전히 동일합니다. 파이썬이 함수 정의 직후 자동으로 데코레이터를 적용합니다.
중급

인수를 받는 함수에 데코레이터 적용

wrapper가 원본 함수의 인수를 그대로 전달해야 합니다:

def log_call(func):
    def wrapper(*args, **kwargs):       # 모든 인수 수용
        print(f"[호출] {func.__name__}({args}, {kwargs})")
        result = func(*args, **kwargs)  # 원본 함수 실행
        print(f"[반환] {result}")
        return result                   # 반환값도 전달!
    return wrapper

@log_call
def add(a, b):
    return a + b

@log_call
def greet(name, greeting="안녕"):
    return f"{greeting}, {name}!"

add(3, 5)
# [호출] add((3, 5), {})
# [반환] 8

greet("철수", greeting="Hello")
# [호출] greet(('철수',), {'greeting': 'Hello'})
# [반환] Hello, 철수!
return result를 빠뜨리면: wrapper가 None을 반환합니다. 원본 함수의 반환값이 사라집니다. 항상 return result를 포함하세요.

실용적인 예: 타이머 데코레이터

import time

def timer(func):
    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        end = time.perf_counter()
        print(f"{func.__name__} 실행시간: {end-start:.4f}초")
        return result
    return wrapper

@timer
def slow_sum(n):
    return sum(range(n))

slow_sum(10_000_000)
# slow_sum 실행시간: 0.1234초
고급

데코레이터 적용 시점

데코레이터는 함수 정의 시(import 시)에 실행됩니다. 호출 시가 아닙니다.

def register(func):
    print(f"등록됨: {func.__name__}")  # 정의할 때 실행!
    return func

@register
def func_a(): pass

@register
def func_b(): pass

print("--- 여기서부터 호출 ---")
func_a()
func_b()

# 출력:
# 등록됨: func_a      ← 정의할 때!
# 등록됨: func_b      ← 정의할 때!
# --- 여기서부터 호출 ---
활용: Flask, Django에서 @app.route("/path")는 함수를 라우팅 테이블에 등록하는 데코레이터입니다. import 시점에 모든 경로가 등록됩니다.
데코레이터가 반환값을 바꾸지 않는 데코레이터는 어떻게 만드나요?
원본 함수를 그대로 반환하면 됩니다: 
registry = []

def register(func):
    registry.append(func)

def retry(max_tries=3, delay=1.0, exceptions=(Exception,)):
    def decorator(func):
        @wraps(func)
        def wrapper(*args, **kwargs):
            for attempt in range(1, max_tries + 1):
                try:
                    return func(*args, **kwargs)
                except exceptions as e:
                    if attempt == max_tries:
                        raise
                    print(f"시도 {attempt}/{max_tries} 실패: {e}. {delay}초 후 재시도...")

        return wrapper
    return decorator

Part A: 클로저 체인과 cell 공유

Part A: 3단 중첩에서 times는 decorator의 자유 변수이고, func는 wrapper의 자유 변수입니다. 즉, wrapper.__closure__에는 두 cell이 있습니다: func에 대한 cell과 times에 대한 cell(decorator가 공유). wrapper.__code__.co_freevars == ('func', 'times')로 확인 가능합니다.

Part B: 왜 Python의 데코레이터는 GoF보다 간결한가

Gang of Four(1994)의 Decorator Pattern은 객체에 동적으로 기능을 추가하는 패턴입니다. Java에서 이를 구현하려면:

  1. 공통 인터페이스를 정의
  2. Wrapper 클래스가 그 인터페이스를 구현
  3. 원본 객체를 Wrapper에 전달
  4. Wrapper가 원본을 호출하면서 추가 동작 수행

Java에서 이것은 인터페이스 + 클래스 + 생성자 + 위임 메서드로 20줄 이상입니다. Python에서는 함수가 First-Class Object이므로 클로저 3줄로 동일한 패턴을 구현합니다.

이것이 가능한 근본적 이유: Java에서 함수는 독립적으로 존재할 수 없고 반드시 클래스에 속해야 합니다. Python에서 함수는 객체이므로 변수에 담고, 인수로 넘기고, 반환할 수 있습니다. GoF 패턴이 클래스 계층으로 해결하는 것을 Python은 함수 합성으로 해결합니다. 이것이 "Python에서는 디자인 패턴이 보이지 않는다"고 말하는 이유입니다.

클래스를 데코레이터로 사용하는 이유

from functools import wraps

class CallCounter:
    """함수 호출 횟수를 세는 클래스 데코레이터"""

    def __init__(self, func):
        wraps(func)(self)       # 메타데이터 복사
        self.func = func
        self.call_count = 0

    def __call__(self, *args, **kwargs):
        self.call_count += 1
        print(f"[{self.call_count}번째 호출]")
        return self.func(*args, **kwargs)

@CallCounter
def greet(name):
    return f"안녕, {name}!"

print(greet("철수"))   # [1번째 호출] 안녕, 철수!
print(greet("영희"))   # [2번째 호출] 안녕, 영희!
print(greet.call_count)  # 2  ← 클래스 인스턴스이므로 속성 접근 가능!
클래스 vs 함수 데코레이터 선택: 상태 저장이 필요하면 클래스 기반, 단순 변환이면 함수 기반이 일반적입니다. 그러나 nonlocal로 상태를 클로저에 담는 것도 동등한 선택입니다.

class Circle:
    def __init__(self, radius):
        self._radius = radius   # _는 "내부용"을 나타내는 관례

    @property
    def radius(self):           # getter
        return self._radius

    @radius.setter
    def radius(self, value):    # setter
        if value < 0:
            raise ValueError(f"반지름은 0 이상이어야 합니다: {value}")
        self._radius = value

    @radius.deleter
    def radius(self):           # deleter
        del self._radius

    @property
    def area(self):             # 읽기 전용 (setter 없음)
        import math
        return math.pi * self._radius ** 2

c = Circle(5)

@property는 Descriptor다

property는 클래스입니다. @property는 그 클래스의 인스턴스를 생성합니다. property 객체는 __get__, __set__, __delete__를 구현한 Descriptor입니다. 

# @property의 수동 등가물
class Circle:
    def __init__(self, radius):
        self._radius = radius

    def get_radius(self):
        return self._radius

    def set_radius(self, value):
        if value < 0: raise ValueError()
        self._radius = value

    radius = property(get_radius, set_radius)
    # radius는 property 인스턴스 (descriptor)

c = Circle(5)

Part A: c.radius 접근 시 Python의 attribute lookup 과정:
  1. type(c).__mro__에서 radius 탐색
  2. 발견된 객체가 Data Descriptor(__get__ + __set__ 둘 다)면 descriptor 우선
  3. property.__get__(c, type(c)) 호출 → getter 실행
property는 Data Descriptor이므로 인스턴스 __dict__보다 우선합니다. Non-data Descriptor(__get__만)라면 인스턴스 dict가 우선합니다.

Part B: Python의 속성 접근 우선순위

obj.attr 접근 시 우선순위: Data Descriptor > instance.__dict__ > Non-data Descriptor > __getattr__. 이 규칙은 Python Data Model의 핵심이며, property, classmethod, staticmethod 모두 descriptor로 구현됩니다.

def timer(func):

    def wrapper(*args, **kwargs):
        start = time.perf_counter()
        result = func(*args, **kwargs)
        elapsed = time.perf_counter() - start
        print(f"⏱ {func.__name__}: {elapsed:.4f}초")

def simple_cache(func):
    cache = {}
    @wraps(func)
    def wrapper(*args):
        if args not in cache:
            cache[args] = func(*args)
        return cache[args]
    wrapper.cache = cache   # 캐시 접근 가능하게
    return wrapper

            if isinstance(arg, (int, float)) and arg < 0:
                raise ValueError(f"양수가 필요합니다: {arg}")
        return func(*args, **kwargs)
    return wrapper

문제 10 (고급): lru_cache 직접 구현
@lru_cache와 유사하게 동작하는 @memoize를 직접 구현하세요 (maxsize 없음). 
from functools import wraps

def memoize(func):
    cache = {}
    @wraps(func)
    def wrapper(*args):
        if args not in cache:
            cache[args] = func(*args)
        return cache[args]
    wrapper.cache_clear = lambda: cache.clear()
    wrapper.cache_info = lambda: {"size": len(cache), "keys": list(cache.keys())}
    return wrapper

@memoize
def fib(n):
    if n < 2: return n
    return fib(n-1) + fib(n-2)

print(fib(35))           # 9227465
print(fib.cache_info())  # {'size': 36, 'keys': [(0,), (1,), (2,), ..., (35,)]}

문제 13 (전공자): Descriptor로 @property 재구현
property 내장 클래스를 모방하는 MyProperty descriptor를 작성하세요. 
class MyProperty:
    def __init__(self, fget=None, fset=None, fdel=None):
        self.fget = fget
        self.fset = fset
        self.fdel = fdel

    def __get__(self, obj, objtype=None):
        if obj is None:
            return self          # 클래스에서 접근: descriptor 반환
        if self.fget is None:
            raise AttributeError("읽기 불가")
        return self.fget(obj)

    def __set__(self, obj, value):
        if self.fset is None:
            raise AttributeError("쓰기 불가")
        self.fset(obj, value)

    def __delete__(self, obj):
        if self.fdel is None:
            raise AttributeError("삭제 불가")
        self.fdel(obj)

    def setter(self, fset):
        return MyProperty(self.fget, fset, self.fdel)

class Temperature:
    def __init__(self, c):
        self._c = c

    celsius = MyProperty(
        fget=lambda self: self._c,
        fset=lambda self, v: setattr(self, '_c', v)
    )

    @MyProperty
    def fahrenheit(self):
        return self._c * 9/5 + 32

t = Temperature(100)
print(t.celsius)     # 100
print(t.fahrenheit)  # 212.0
t.celsius = 0
print(t.fahrenheit)  # 32.0
