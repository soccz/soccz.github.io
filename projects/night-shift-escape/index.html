Overview

프로젝트 목표

외부 게임 엔진을 사용하지 않고 브라우저 API만으로 완성도 있는 게임을 만들 수 있는지 검증하는 것이 목표였습니다. Canvas 2D 렌더링, 경로 탐색, 다중 AI, 오디오 합성을 직접 구현하며 각 시스템이 어떻게 상호작용하는지 설계하는 데 초점을 뒀습니다.

No Framework
순수 Vanilla JS + Canvas
React, Phaser, PixiJS 없이 requestAnimationFrame 기반 게임 루프를 직접 구현했습니다. 빌드 도구도 없이 단일 app.js로 전체 게임 로직을 관리합니다.
State Design
3-객체 상태 분리
game (런 상태), state (UI·메타), world (정적 맵)으로 나눠 책임을 분리했습니다. 각 서브시스템은 이 객체를 통해서만 통신하며 직접 참조를 최소화합니다.
Scalability
데이터 주도 설계
Run Profile, Ward Event, Anomaly, Contract는 모두 데이터 배열로 정의됩니다. 새 이벤트 타입을 추가할 때 렌더링·업데이트 코드를 건드리지 않아도 됩니다.

경로 탐색 (Pathfinding)

방과 복도를 노드로 연결한 정적 내비게이션 그래프 위에서 A*를 실행합니다. 층(f1/f2) 간 연결은 엘리베이터 노드로만 이어지며, 헌터 AI는 이 그래프를 통해 목표 방까지의 경로를 계산합니다.

// 헌터가 목표 방 문 앞으로 이동하는 흐름
function hunterSeekPoint(x, y) {
  hunter.targetX = x;
  hunter.targetY = y;
}

function moveEntityWithPath(entity, tx, ty, amount) {
  const wp = getNextWaypoint(entity, tx, ty);  // A* 다음 웨이포인트
  if (wp) moveTowardTarget(entity, wp.x, wp.y, amount);
  else    moveTowardTarget(entity, tx, ty, amount);  // fallback: 직선 이동
}

절차적 오디오 (Web Audio API)

음악 파일 없이 오실레이터를 실시간으로 합성합니다. 이벤트 종류에 따라 주파수·파형·볼륨·지속 시간을 달리하여 다양한 음향 피드백을 만듭니다.

function playUiTone(freq, dur, type, vol) {
  const osc = ctx.createOscillator();
  const gain = ctx.createGain();
  osc.type = type;          // "sine" | "sawtooth" | "square" | "triangle"
  osc.frequency.value = freq;
  gain.gain.setValueAtTime(vol, ctx.currentTime);
  gain.gain.exponentialRampToValueAtTime(0.001, ctx.currentTime + dur);
  osc.connect(gain).connect(ctx.destination);
  osc.start(); osc.stop(ctx.currentTime + dur);
}

// 헌터 발소리: 근접할수록 주기 짧게, 저주파 sawtooth
if (dist < 320 && sameFloor) {
  const interval = 0.4 + (dist / 320) * 0.6;
  if (footstepTimer <= 0) { playUiTone(60, 0.08, "sawtooth", 0.025); footstepTimer = interval; }
}

돌연변이 시계 & 방 균열

mutationClock은 플레이어가 한 방에 머무는 시간을 누적합니다. Run Profile의 mutationRate 배수가 적용되며, 임계값(75% 도달 시 경고, 100% 시 균열)을 초과하면 방에 사이드 패스가 열려 주기적으로 감염자가 스폰됩니다. 방을 바꾸면 시계가 감소합니다.

Run Profile 시스템

매 런 시작 시 10여 종의 Omen 중 하나가 선택되어 runProfile.modifiers 객체를 구성합니다. 모든 수치(감염자 HP, 헌터 속도, 골드 수입, 워드 이벤트 주기 등)는 이 배수 객체에서 읽히므로, 별도의 난이도 분기 없이 전체 시스템에 동시에 적용됩니다.

Engineering Challenges

구현 중 해결한 문제들

단일 파일 구조에서 복잡도가 높아질수록 상태 간섭, 타이밍 오류, AI 불일치 등이 누적됩니다. 10라운드 코드 감사를 통해 발견하고 수정한 주요 문제들입니다.

State Leak
페이즈 간 상태 누수
탈출/실패 페이즈 진입 후에도 일부 타이머(decoyTimer 만료 로그, wardEvent 효과)가 실행되는 문제. 모든 이벤트 발화 지점에 game.phase === "running" 가드 추가로 해결.
Timer Drift
탭 전환 시 dt 폭주
백그라운드 탭에서 돌아올 때 누적된 Δt가 한 프레임에 반영되어 타이머들이 한꺼번에 0을 지나치는 문제. Math.max(0, Math.min(0.033, Δt/1000))로 프레임 dt를 33ms로 클램핑.
AI Cross-Floor
레이드 시 헌터 층 불일치
레이드가 발동될 때 헌터가 점거 방과 다른 층에 있으면 navGraph에 층 간 연결이 없어 경로 탐색이 실패. 레이드 발동 시 헌터를 목표 층 스폰 포인트로 텔레포트 후 enrage 모드 진입.
Orphan Units
방 상실 시 유닛 고아
방이 감염되거나 하이더가 패닉 탈출할 때 game.units 정리를 누락, 소속 없는 가디언이 잔존하는 문제. 방 소유권 변경 3개 경로(abandonRoom, 감염, 패닉) 모두에 filter 추가.
Reclaim Bug
방 재점거 시 손상 상태 유지
플레이어가 방을 포기했다가 재점거할 때 이전 런에서 손상된 문 HP, breach 플래그가 초기화되지 않는 문제. claimRoom()에서 door.hp = door.maxHp, breach = false 명시 리셋.
Stuck Keys
포커스 이탈 시 키 고착
창이 포커스를 잃을 때 keyup 이벤트가 발화되지 않아 키가 Set에 남아 플레이어가 계속 이동하는 문제. window.blur 시 keys.clear()로 해결.
Audit Process

10가지 관점(상태 관리, 헌터 AI, 유닛 시스템, 상점·계약, 렌더링, 탈출 흐름, 오디오, 골드·메타, 이상 현상, 입력 처리)으로 10라운드 병렬 코드 감사를 수행해 총 30여 개의 버그를 발견·수정했습니다.
